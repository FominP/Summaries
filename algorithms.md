# Грокаем алгоритмы
Это книга об алгоритмах в программировании. В ней вводится понятие алгоритмов, приводится описание принципа работы для основных из них, объясняются различия между структурами данных. Книга важна для обучения программированию.

## Основные идеи

### Сложность алгоритмов
- Для O-большого log это log2
- O - это операции
- O-большое определяет время в худшем случае
- Константы не влияют на сложность

### Связанный список
- При использовании связанного списка элементы могут занимать любую ячейку памяти
- В каждом элементе хранится адрес следующего
- Массивы занимают блоки в памяти, а не любые места
- Связанные списки читаются последовательно. Их удобно использовать, когда операций вставки больше, особенно в середину и конец массива. Удаление тоже быстрее
- **Указатель** - адрес элемента в связанном списке

Доступ к элементам бывает:
1) Произвольный
2) Последовательный

**Сортировка выбором** - обычная сортировка за O(n^2) с выбором самого большого значения каждый раз.

**Рекурсия** - вызов функцией самой себя для упрощения читаемости. В рекурсии приводится базовый и рекурсивный случай.

``` python
def countdown(i):
    print(i)
    if i<=1:
        return #базовый случай
    else:
        coiuntdown(i-1) #рекурсивный случай
```

### Стек
- Стек используется для управления памятью при работе с функциями, при вызове функции внутри другой функции под новую выделяется место в памяти, которое очищается после выполнения внутренней функции
- Доступ к элементам последовательный по принципу **LIFO** (last in, first out)
- Стек важен для рекурсии
- Стек может занимать много памяти
- Стек поддерживает только внесение и извлечение элементов

### Решение методом "разделяй и властвуй"
1) Определяется базовый случай - самый простейший из всех возможных *(в задаче на лампочки в поезде это ситуация с 1 вагоном)*
2) Задача делится и сокращается, пока не сведётся к базовому случаю

Если рекурсивная функция работает с массивом, то базовым случаем часто является массив из 1 элемента или пустой массив

### Быстрая сортировка
- Быстрая сортировка - рекурсивный алгоритм, базовый случай это 1 элемент
- Берется опорный элемент, и массив делится на 2 части: всё, что меньше опорного элемента, и всё, что больше
- Быстродействие быстрой сортировки зависит от выбора элемента
- Если массив отсортирован, то быстрая сортировка имеет сложность O(n x log(n)), так как высота стека вызовов равна n x log(n) уровней: она обращается к log(n) элементов, а не к n как в несортированном массиве. Можно выбрать случайный элемент - всё равно время выполнения будет O(n x log(n))

### Хеш-функция
- Хеш-функция отражает строки на числа
- Словари в Python - это хеш-таблица
- Хеш-функция последовательна. Последовательная функция возвращает один и тот же результат для одинаковых входных данных
- Преобразование DNS - препобразование веб-адреса сайта в ip-адрес. Для кэширования веб-страниц тоже используется хеш
- В случае, когда элементы отражаются на один и тот же индекс (коллизия), в нём создаётся связанный список. Чтобы избежать длинных связанных списков, нужно выбрать хорошую хеш-функцию.
- **Simhash** - локально-чувствительная хеш-функция, то есть строки можно сравнивать между собой (если строки были похожи, то и хеши будут похожи)

### Графы
- Алгоритм для нахождения кратчайшего пути называется поиском в ширину
- При поиске в ширину важно вести учёт проверенных элмементов, чтобы не попасть в бесконечный цикл
- Граф состоит из узлов и рёбер. Точки - это узлы, соединяются они рёбрами.
- Если граф выглядит как A->B, то A и B это узлы, между ними ребро, A - внешний сосед B, B - внутренний сосед A

``` python
graph = {}
graph['A'] = ['B', 'C', 'D']
graph['C'] = ['E', 'F']
graph['B'] = ['F']
graph['D'] = ['G', 'H']
graph['E'] = []
graph['F'] = []
graph['G'] = []
graph['H'] = []
```

### Очередь
Очередь - структура данных, где доступны только постановка в очередь и извлечение из очереди по принципу **FIFO** (first in, first out)
Для создания очереди в Python есть дека:

``` python
from collections import deque
search_queue = deque()
search_queue += graph['A']

queue.popleft() #вынуть элемент из очереди
```

### Дерево
- Дерево - граф, в котором нет рёбер, указывающих в обратном направлении
- Дерево - связанный циклический граф
- Поиск в ширину проходит все элементы дерева. Поиск в глубину проверяет все элементы поочерёдно до конца (используется стек)
- В деревьях не бывает циклов
- Деревья, где от узла исходит только 2 ребра, называются бинарными
- **Сбалансированное бинарное дерево поиска - BST** - в котором левая часть дерева всегда больше правого по значению. Поиск и вставка в BST-деревья проходит за O(log(n))
- **АВЛ-дерево** - автоматически сбалансированное дерево. Можно получить поворотом прямого однонаправленного графа с 1 ребром для каждого узла. Его высота станет log(n) элементов вместо n -> ускорится поиск и вставка элементов

### Алгоритм Дейкстры
1) Найти узел с наименьшей стоимостью
2) Обновить стоимость соседей
3) Повторять, пока не будет сделано для всех узлов графа
4) Вычислить итоговый путь
- Подходит только для графов с неотрицательными весами (для этого алгоритм Беллмана-Форда)

### Жадные алгоритмы
- Жадный алгоритм на каждом шаге выбирает локально-оптимальное решение
- Если имеется NP-трудная задача, то лучше воспользоваться приближенным алгоритмом. Жадные алгоритмы часто для этого подходят

**Закон Амдала** - при оптимизации одной части системы выигрыш в производительности ограничивается тем, какую долю времени от общего процесса занимает эта часть.

Типы задач:
1) Задачи разрешимости (ответ "да" или "нет")
2) NP-задачи (быстрее проверить, чем решить)
3) P - быстро решить, быстро проверить
